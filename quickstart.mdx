## Quick setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/runpod/poddy.git
    cd poddy
    ```
  </Step>

  <Step title="Install dependencies">
    The Prisma client is generated automatically during installation:
    
    ```bash
    pnpm install
    ```
    
    <Note>
      This will install all dependencies and automatically run `prisma generate` to create the database client.
    </Note>
  </Step>

  <Step title="Configure environment">
    Copy the example environment file and fill in your values:
    
    ```bash
    cp .env.example .env.dev
    ```
    
    Edit `.env.dev` with your configuration:
    
    <CodeGroup>
    ```bash .env.dev
    # Required Discord credentials
    DISCORD_TOKEN="your-bot-token"
    APPLICATION_ID="your-application-id"
    
    # Development guild for testing slash commands
    DEVELOPMENT_GUILD_ID="your-test-server-id"
    
    # Database connection (matches docker-compose.yml)
    DATABASE_URL="postgresql://poddy:poddy_dev_password@localhost:5432/poddy_dev"
    
    # HTTP server port
    API_PORT=3423
    
    # Optional but recommended
    SENTRY_DSN=""
    DATADOG_API_KEY=""
    RUNPOD_API_KEY=""
    
    # AI Q&A (optional)
    MASTRA_AGENT_URL=""
    RUNPOD_ASSISTANT_API_KEY=""
    
    # Webhooks (optional)
    CHANGELOG_WEBHOOK_SECRET=""
    
    # Data export (optional)
    EXPORTER_BUCKET_NAME=""
    EXPORTER_FILE_PATH=""
    EXPORTER_ACCESS_KEY=""
    EXPORTER_SECRET_KEY=""
    ```
    </CodeGroup>
    
    <Warning>
      The `DATABASE_URL` must match the credentials in `docker-compose.yml` for local development.
    </Warning>
  </Step>

  <Step title="Start the database">
    Launch PostgreSQL using Docker Compose:
    
    ```bash
    pnpm db:start
    ```
    
    This starts a PostgreSQL 16 container with:
    - **Container name**: `poddy-postgres`
    - **Port**: 5432
    - **User**: `poddy`
    - **Password**: `poddy_dev_password`
    - **Database**: `poddy_dev`
  </Step>

  <Step title="Apply database migrations">
    Create the database schema:
    
    ```bash
    pnpm db:migrate
    ```
    
    This runs all Prisma migrations to set up tables for:
    - Command cooldowns
    - User preferences and languages
    - Auto-threading and auto-tagging
    - Custom embeds and tags
    - Event management and submissions
    - Zendesk ticket tracking
    - Logging and audit trails
    - Better Stack incident reports
  </Step>

  <Step title="Build and run the bot">
    Compile TypeScript and start the bot in development mode:
    
    ```bash
    pnpm build
    ```
    
    <Note>
      Despite the name, `pnpm build` both compiles **and** runs the bot. It's not just a build step!
    </Note>
    
    The bot will:
    1. Compile TypeScript to the `dist/` directory
    2. Resolve path aliases using `tsc-alias`
    3. Start in development mode (guild commands only)
    4. Register slash commands to your development guild
    5. Connect to Discord gateway with worker sharding
  </Step>
</Steps>

## Verify it's working

Once the bot starts, you should see:

```bash
[INFO] HTTP server started on port 3423
[INFO] All shards have started.
[DEBUG] [SHARD 0] Ready
```

In your Discord server:
1. The bot should appear online
2. Slash commands should be available (guild commands only in dev mode)
3. Check the bot's presence status

<Check>
  Congratulations! Poddy is now running. Try using a slash command in your development server.
</Check>

## Development vs production modes

Poddy has two distinct run modes:

<Tabs>
  <Tab title="Development mode">
    **Command**: `pnpm build`
    
    - Uses `.env.dev` configuration
    - Registers **guild commands** (instant updates)
    - Debug logging enabled
    - Points to `api.runpod.dev`
    - Ideal for testing new features
  </Tab>
  
  <Tab title="Production mode">
    **Command**: `pnpm start`
    
    - Uses `.env.prod` configuration
    - Registers **global commands** (can take up to 1 hour to propagate)
    - Standard logging
    - Points to `api.runpod.io`
    - Used for the live RunPod Discord bot
    
    <Warning>
      Production mode requires pre-compiled code. Run `pnpm build` first to compile, then stop the bot and run `pnpm start`.
    </Warning>
  </Tab>
</Tabs>

## Common issues

<AccordionGroup>
  <Accordion title="Database connection failed">
    **Error**: `Can't reach database server at localhost:5432`
    
    **Solutions**:
    1. Ensure PostgreSQL is running: `docker ps | grep poddy-postgres`
    2. Start the database: `pnpm db:start`
    3. Verify `DATABASE_URL` in `.env.dev` matches the docker-compose.yml credentials
  </Accordion>
  
  <Accordion title="Type error: 'K' cannot be used to index type 'LanguageValues'">
    **Cause**: Language type definitions are out of sync.
    
    **Solution**: Regenerate language typings:
    ```bash
    pnpm translate
    ```
  </Accordion>
  
  <Accordion title="Bot is offline in Discord">
    **Possible causes**:
    1. Invalid `DISCORD_TOKEN` in `.env.dev`
    2. Bot token doesn't match the `APPLICATION_ID`
    3. Insufficient bot permissions/intents in Discord Developer Portal
    
    **Solution**: Verify your bot configuration in the [Discord Developer Portal](https://discord.com/developers/applications).
  </Accordion>
  
  <Accordion title="Slash commands not appearing">
    **In development**:
    - Commands are registered to `DEVELOPMENT_GUILD_ID` only
    - Verify the guild ID is correct
    - Commands should appear instantly after bot starts
    
    **In production**:
    - Global commands can take up to 1 hour to propagate
    - Be patient or use development mode for testing
  </Accordion>
  
  <Accordion title="Port 3423 already in use">
    **Solution**: Change `API_PORT` in `.env.dev` to an available port:
    ```bash
    API_PORT=3424
    ```
  </Accordion>
</AccordionGroup>

## Useful commands

<CodeGroup>
```bash Database
# Start PostgreSQL
pnpm db:start

# Stop PostgreSQL
pnpm db:stop

# Reset database (deletes all data)
pnpm db:reset

# Apply schema changes
pnpm db:migrate

# Open Prisma Studio (GUI)
pnpm db:studio
```

```bash Development
# Build and run in dev mode
pnpm build

# Run in production mode
pnpm start

# Regenerate language typings
pnpm translate

# Format and lint code
pnpm biome check --write
```
</CodeGroup>

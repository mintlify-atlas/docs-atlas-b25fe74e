## What is Poddy?

Poddy is a powerful Discord bot built specifically for the RunPod community server. It streamlines support operations, enhances community engagement, and provides intelligent assistance through AI-powered Q&A.

## Architecture overview

Poddy is built on modern Discord infrastructure using the official Discord.js libraries:

- **Core**: `@discordjs/core`, `@discordjs/rest`, `@discordjs/ws`
- **Database**: PostgreSQL with Prisma ORM
- **Runtime**: Node.js 24 with TypeScript
- **Package manager**: pnpm v10
- **HTTP server**: Hono for webhooks and API endpoints

<Note>
  Poddy uses the low-level Discord.js libraries instead of the high-level discord.js framework, providing better performance and more control over bot behavior.
</Note>

## Tech stack

<AccordionGroup>
  <Accordion title="Discord libraries">
    - `@discordjs/core` - Core Discord API wrapper
    - `@discordjs/rest` - REST API client
    - `@discordjs/ws` - WebSocket gateway manager with worker sharding
  </Accordion>
  
  <Accordion title="Database and ORM">
    - PostgreSQL 16 (via Docker)
    - Prisma ORM for type-safe database access
    - Automated migrations and schema management
  </Accordion>
  
  <Accordion title="Integrations">
    - **Zendesk** - Support ticket creation and management
    - **BetterStack** - Incident monitoring and status reporting
    - **Mastra** - AI-powered documentation Q&A
    - **Sentry** - Error tracking and monitoring
    - **Datadog** - Metrics and performance monitoring
  </Accordion>
  
  <Accordion title="Developer tooling">
    - Biome for formatting and linting (120-char line width, tabs)
    - Husky + lint-staged for pre-commit hooks
    - TypeScript with strict type checking
    - Path aliases for clean imports
  </Accordion>
</AccordionGroup>

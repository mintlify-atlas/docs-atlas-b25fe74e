This page documents the complete database schema for Poddy, including all models, fields, enums, and relations.

## Enums

### CommandType

Defines the type of command that triggered a cooldown.

| Value | Description |
|-------|-------------|
| `TEXT_COMMAND` | Legacy text-based commands (prefix commands) |
| `APPLICATION_COMMAND` | Slash commands and context menu commands |

### EscalatedType

Defines what type of Discord content was escalated to Zendesk.

| Value | Description |
|-------|-------------|
| `MESSAGE` | A single Discord message was escalated |
| `THREAD` | An entire Discord thread was escalated |

### LogEvent

All possible events that can be logged to configured log channels.

| Value | Description |
|-------|-------------|
| `MESSAGE_EDITED` | A message was edited |
| `MESSAGE_DELETED` | A message was deleted |
| `CHANNEL_CREATED` | A channel was created |
| `CHANNEL_DELETED` | A channel was deleted |
| `CHANNEL_UPDATED` | A channel was updated |
| `ROLE_CREATED` | A role was created |
| `ROLE_DELETED` | A role was deleted |
| `ROLE_UPDATED` | A role was updated |
| `MEMBER_ROLE_UPDATED` | A member's roles were changed |
| `MEMBER_UPDATED` | A member was updated |
| `MEMBER_JOINED` | A member joined the server |
| `MEMBER_LEFT` | A member left the server |
| `VOICE_STATE_UPDATE` | A member's voice state changed |
| `THREAD_CREATED` | A thread was created |
| `THREAD_UPDATED` | A thread was updated |
| `THREAD_DELETED` | A thread was deleted |
| `INVITE_CREATED` | An invite was created |
| `INVITE_DELETED` | An invite was deleted |
| `INCIDENT_CREATED` | A BetterStack incident was created |

## Models

### Cooldown

Stores command cooldown state for rate limiting.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String | Discord user ID |
| `commandName` | String | Name of the command |
| `expiresAt` | DateTime | When the cooldown expires |
| `commandType` | CommandType | Type of command (text or application) |

**Composite Primary Key:** `[commandName, commandType, userId]`

**Table Name:** `command_cooldowns`

### UserLanguage

Stores user language preferences.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String | Discord user ID (Primary Key) |
| `languageId` | String | Language code (e.g., "en-US") |

**Table Name:** `user_languages`

### AutoThreadChannel

Channels configured to automatically create threads for new messages.

| Field | Type | Description |
|-------|------|-------------|
| `channelId` | String | Discord channel ID (Primary Key) |
| `guildId` | String | Discord guild ID |
| `threadName` | String? | Optional custom thread name pattern |

**Table Name:** `auto_thread_channels`

### AutoTagOnForumChannel

Tags that are automatically applied to new posts in forum channels.

| Field | Type | Description |
|-------|------|-------------|
| `channelId` | String | Discord channel ID |
| `guildId` | String | Discord guild ID |
| `tagId` | String | Discord tag ID |

**Composite Primary Key:** `[channelId, tagId]`

**Table Name:** `auto_tag_on_forum_channels`

### Embed

Stored embed templates that can be reused.

| Field | Type | Description |
|-------|------|-------------|
| `id` | String | CUID identifier (Primary Key, auto-generated) |
| `name` | String | Embed template name |
| `data` | String | JSON string of embed data |
| `guildId` | String | Discord guild ID |

**Unique Constraint:** `[name, guildId]`

**Table Name:** `embeds`

### Event

Community events with submissions and code generation.

| Field | Type | Description |
|-------|------|-------------|
| `id` | String | CUID identifier (Primary Key, auto-generated) |
| `guildId` | String | Discord guild ID |
| `channelId` | String | Channel where event is hosted |
| `name` | String | Event name |
| `active` | Boolean | Whether event is currently active (default: true) |
| `description` | String? | Optional event description |
| `codeAmount` | Int? | Optional number of codes to generate |
| `codeLogChannelId` | String? | Optional channel for code generation logs |

**Unique Constraint:** `[name, guildId]`

**Relations:**
- `submissions` - One-to-many with Submission
- `generatedCodes` - One-to-many with GeneratedCode

### GeneratedCode

Codes generated for event participants.

| Field | Type | Description |
|-------|------|-------------|
| `code` | String | The generated code (Primary Key) |
| `eventId` | String | Related event ID |
| `userId` | String | Discord user ID who received the code |
| `runpodEmail` | String | User's Runpod email address |

**Unique Constraints:**
- `[runpodEmail, eventId]`
- `[userId, eventId]`

**Relations:**
- `event` - Many-to-one with Event (cascades on delete)

**Table Name:** `generated_codes`

### Submission

User submissions to events.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String | Discord user ID |
| `eventId` | String | Related event ID |
| `messageId` | String | Discord message ID (unique) |

**Composite Primary Key:** `[userId, eventId]`

**Relations:**
- `event` - Many-to-one with Event (cascades on delete)
- `upvotes` - One-to-many with SubmissionUpvote

### SubmissionUpvote

Upvotes on event submissions.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String | Discord user ID who upvoted |
| `submissionId` | String | Message ID of the submission |
| `eventId` | String | Related event ID |

**Composite Primary Key:** `[userId, submissionId]`

**Unique Constraint:** `[userId, eventId]` (one upvote per event per user)

**Relations:**
- `submission` - Many-to-one with Submission (cascades on delete)

### ZendeskTicket

Tracks Discord messages/threads escalated to Zendesk.

| Field | Type | Description |
|-------|------|-------------|
| `id` | Int | Zendesk ticket ID (Primary Key) |
| `escalatedId` | String | Discord message or thread ID (unique) |
| `escalatedById` | String | Discord user ID who escalated |
| `type` | EscalatedType | Whether a message or thread was escalated |

**Table Name:** `zendesk_tickets`

### LogChannel

| Field | Type | Description |
|-------|------|-------------|
| `guildId` | String | Discord guild ID |

## Database Configuration

The schema uses PostgreSQL as the database provider:

```prisma
datasource db {
  provider = "postgresql"
}
```

The Prisma client is generated to `prisma/__generated__/`:

```prisma
generator client {
  provider = "prisma-client"
  output   = "./__generated__/"
}
```
